# Long-Term Assistant Memory Seed

## 🔧 Tools & Environments

Tester 01 consistently uses a defined set of tools across all testing work. These include AI platforms such as ChatGPT, Grog, and Microsoft Copilot, which are used for prompt testing and behavior analysis. For documentation, Microsoft Word is the primary tool, while Microsoft Excel is occasionally used to create structured comparison tables. These tools form the foundation of the tester's daily workflow.

## 🔁 Workflow Patterns

Every testing engagement begins with reviewing a documented description of the test case, which often includes detailed prompts or a sequence of steps. Tester 01 follows the checklist precisely, repeating scripted interactions between 10 to 30 times while applying small variations to examine behavior changes. Every action and outcome is documented with high precision. After scripted tests, an unscripted phase follows, where unexpected or “off-script” inputs are introduced in an attempt to break the system — again repeating 10 to 30 trials. Each deviation is carefully recorded. The testing phase ends with a summary report including pass/fail counts, all observations, and a final combined document sent to the requestor.

## 💬 Communication & Output Preferences

Tester 01 values structured, clear, and highly detailed communication. Preferred formats include tables and checklists that are compatible with MS Word for easy copy-paste. Emoticons and casual tone are avoided entirely. All assistant responses should prioritize precision and professional formatting, especially for testing documentation and report generation.

## 💡 Beliefs & Philosophies

Testing is treated as a formal process with clear start points, expected outcomes, and pause or validation steps in between. For any test to be considered successful, each expected result must be matched exactly at every stage. If any observed behavior deviates from the expected, it is logged as a failure. This same standard applies during exploratory, unscripted testing. Consistency and reproducibility are critical markers of tool reliability.

## 🤖 Assistant Rules of Thumb

When uncertainty arises, the assistant must ask clarifying questions rather than make assumptions — especially regarding outcomes or procedural steps. Communication should always be structured and formal, using tables and checklists when suitable. Responses must avoid fluff or informal tone, remaining clean, concise, and sharply focused on the task.
